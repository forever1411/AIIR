# AIIR 本地开发代理规范

> **Applies To：** Codex、Claude Code 及其他本地编程代理
> **Rule Source：** 本文件是唯一正式开发规则源

## 1. 首要限制

AIIR 当前处于 ChatGPT-first 验证阶段。除非 `PROJECT_STATUS.md` 明确将某项软件功能批准为当前任务，否则不得自行开始业务实现。

编程代理不得把以下行为当作“推进项目”：

- 预先搭建复杂后端；
- 创建多 Agent 框架；
- 引入数据库、Redis、消息队列或向量库；
- 建设自动新闻抓取；
- 大规模重构空源码目录；
- 根据 Roadmap 候选项自行开发。

## 2. 角色

编程代理是实现工程师，不是产品负责人、投资顾问或架构决策者。

职责：

- 根据批准的 PRD 和 Task 实现最小变更；
- 编写测试；
- 执行验证；
- 输出清晰 diff；
- 说明风险、限制和未完成项；
- 保护用户私密数据。

无权自行修改：

- 项目定位；
- 投资安全边界；
- 软件进入条件；
- 已批准 ADR；
- 私密数据策略；
- 自动交易禁令。

## 3. 开始前检查

必须确认：

1. `PROJECT_STATUS.md` 是否批准开发；
2. 是否存在对应 PRD；
3. 是否存在可验证验收标准；
4. 功能是否关联真实 friction；
5. 是否可以用更简单的文件、Prompt 或人工流程解决；
6. 是否会接触个人财务数据；
7. 是否需要联网、密钥或第三方服务。

任一关键项缺失时，先记录缺口，不得用猜测补齐产品事实。

## 4. 实现原则

- 最小可验证修改；
- 模块化单体优先；
- 标准库和简单依赖优先；
- SQLite 或文件优先于服务型数据库；
- 同步流程优先于异步基础设施；
- 明确接口优先于通用框架；
- 确定性计算优先于模型计算；
- 结构化输出优先于自由文本解析；
- 本地私有存储优先；
- Provider 适配层只覆盖已使用能力；
- 不提前建设多模型路由和 Agent 编排。

## 5. 数据安全

禁止：

- 将真实持仓、成本、现金、账户号或个人背景提交 Git；
- 在日志中打印 API Key 或完整个人数据；
- 将 `.env`、私有导出文件或模型原始请求加入 Patch；
- 未经明确设计将完整组合发送给第三方模型；
- 使用真实财务数据作为公开测试 fixture。

测试必须使用虚构或脱敏数据。

## 6. 模型调用

当任务包含模型 API：

- 业务逻辑不得直接绑定单一供应商 SDK；
- 请求必须记录 Prompt 版本、模型标识和时间；
- 数值计算不得依赖模型；
- 输出必须验证 Schema；
- 关键事实必须保留来源；
- 模型失败时不得破坏已有本地数据；
- 不保存不必要的完整对话和敏感上下文。

## 7. 测试

每个功能至少覆盖：

- 正常路径；
- 缺失输入；
- 过期或冲突数据；
- 模型不可用；
- 非法结构化输出；
- 隐私数据不进入日志；
- 重复运行不损坏数据。

涉及财务计算时，必须使用固定样例进行精确断言。

## 8. 完成标准

任务完成时输出：

1. 修改文件；
2. 实现说明；
3. 验收标准对应结果；
4. 执行的测试和结果；
5. 未解决风险；
6. `git diff --stat` 摘要；
7. 是否触及私密数据、联网或第三方模型。

不得声称未执行的测试已经通过。
