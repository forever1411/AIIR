## 2. 本轮战略重置

项目已放弃以下默认假设：

- 必须先建设完整投资研究平台；
- 必须模拟一个拥有几十年经验的权威专家；
- 必须具备多 Agent、知识图谱、实时新闻和自动报告；
- 必须通过软件才能开始产生价值；
- 功能越多，AIIR 就越接近目标。

项目采用新的核心假设：

> 成熟大模型已经能完成多数单次投资学习和研究任务。AIIR 只有在主动引导、个人长期上下文、确定性计算、风险约束、决策记录和复盘方面产生可观察增益，才值得开发。

## 3. 当前验证问题

### 主问题

一套 ChatGPT-first 的引导流程，能否帮助没有投资经验的用户：

- 从模糊需求开始；
- 知道下一步该了解什么；
- 找到并理解可靠资料；
- 在决定前识别缺失信息和风险；
- 保存当时的决定理由；
- 在以后进行真实复盘？

### 次问题

- 哪些工作 ChatGPT 已经足够好？
- 哪些问题反复要求用户重新提供上下文？
- 哪些数值或状态不适合依赖聊天记忆？
- 哪些流程必须结构化才能防止遗漏？
- 哪些自动化会减少负担，哪些会制造信息焦虑？

## 5. Validation Cycle 0.1

### 场景 A：从零了解一个投资标的

用户只需要输入类似：

> 我想了解英伟达，但不知道该看什么。

验证系统能否主动组织业务、财务、竞争、估值、风险和资料来源。

### 场景 B：讨论个人资产配置

从目标、期限、现金需求、负债、风险承受能力和当前组合开始，而不是直接给出比例。

### 场景 C：记录并复盘一个真实决定

记录当时已知信息、理由、反方观点、失效条件和计划复盘时间。

## 6. 试点验收标准

至少完成 3 个真实场景，并分别记录：

- 输入是否足够简单；
- 系统是否主动提出了用户不知道的问题；
- 是否找到了可核查来源；
- 是否清楚区分事实、推断和未知；
- 是否发现了关键风险和反方证据；
- 是否比普通聊天减少重复工作；
- 记录是否能在一段时间后支持复盘；
- 用户是否愿意重复使用该流程；
- 哪些摩擦出现至少两次。

## 7. 软件开发进入条件

只有满足以下条件，才创建首个软件 PRD：

1. 试点完成至少三个真实场景；
2. 至少一个摩擦出现两次以上；
3. 该摩擦无法通过改进 Project 指令或模板低成本解决；
4. 软件化后能明确降低时间、错误或认知负担；
5. 功能可以独立交付并在两周内验证；
6. 不要求先建设多 Agent、复杂后端或全量数据平台。

## 8. 当前软件候选池

以下仅是候选，不是承诺：

- 本地持仓与投资背景存储；
- 一键生成模型所需的最小上下文包；
- 官方财报和公告发现；
- 仓位、集中度和资产类别暴露计算；
- 决策记录与版本化复盘；
- 新资料与旧 Thesis 的差异比较；
- 研究来源和数据截止时间校验。

候选只有在摩擦日志中获得证据后才能进入 Roadmap。

## 9. 当前风险

| 风险 | 应对 |
|---|---|
| 把模型专业语气当作可靠性 | 强制来源、不确定性和反方证据 |
| 过早信任个性化建议 | 先核对个人事实和风险边界 |
| 确认偏误 | 强制无持仓视角、反方分析和失效条件 |
| 信息过载和过度交易 | Quiet by Default，只关注实质变化 |
| 个人财务数据泄露 | 私有目录、最小化传输、禁止提交 Git |
| 文档再次膨胀 | 文档必须服务当前验证问题 |
| 为开发而开发 | 未满足进入条件不得创建业务实现任务 |
| 用收益评价短期质量 | 评价流程和决策质量，不评价短期盈亏 |
