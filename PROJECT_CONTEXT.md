# AIIR 项目上下文

> **Document Type：** Stable Context
> **Owner：** Project Owner
> **Last Updated：** 2026-07-22

本文件只保存相对稳定的项目背景。当前阶段、当前任务和完成状态统一见 `PROJECT_STATUS.md`。

## 1. 项目起点

项目负责人投资经验有限，希望有人能够像一位耐心、系统、长期在线的财经教练一样：

- 告诉自己现在应该学习什么；
- 在不知道如何提问时主动补全问题；
- 找到财报、新闻、公告和可靠数据；
- 解释这些材料为什么重要；
- 帮助理解资产配置和风险；
- 在作出决定前指出还缺什么；
- 记录当时的理由，并在以后复盘。

最初，AIIR 被定义为一个能够完成所有投资研究工作的专业 AI 平台。随着讨论深入，项目确认：成熟大模型已经能够完成大量检索、解释和分析工作，从零重复建设这些能力既昂贵，也可能比直接使用成熟工具更差。

## 2. 重新定义的问题

AIIR 不再试图回答：

> 怎样制造一个拥有几十年经验、能够替用户给出可靠投资答案的 AI 专家？

AIIR 现在试图回答：

> 怎样让一个投资经验有限的人，能够借助当前最好的成熟模型，逐步进入一个可靠、主动引导、风险优先、可追溯和可复盘的投资学习与决策流程？

## 3. 核心判断

### 3.1 成熟模型是能力来源，不是竞争对手

AIIR 不训练基础模型，也不以单次分析超过 ChatGPT 为目标。检索、解释、综合和讨论优先交给当前成熟模型。

### 3.2 AIIR 的潜在价值来自控制层

AIIR 只有在以下方面提供稳定增益时才值得软件化：

- 准确保存个人投资事实；
- 在用户不知道问什么时主动组织流程；
- 固化不可跳过的风险检查；
- 对数值和仓位进行确定性计算；
- 保存决策时点和理由；
- 对比新事实与旧判断；
- 过滤不重要信息；
- 支持长期复盘；
- 降低重复劳动和上下文搬运。

### 3.3 项目首先是一项验证实验

在开发软件前，先用 ChatGPT Project、结构化文件和人工流程运行未来的 AIIR。只有经过真实使用证明存在稳定缺口，才开发对应功能。

## 4. 目标用户

当前及可预见阶段唯一用户是项目负责人本人。

项目不为以下目标预先设计：

- 公共发布；
- 商业化；
- 多用户或多租户；
- 机构工作流；
- 法规覆盖多个司法辖区；
- 社交、社区或内容平台；
- 券商连接和自动交易。

## 5. 用户当前状态

已知稳定事实：

- 用户希望改善个人理财与投资决策；
- 用户投资经验有限；
- 用户并不知道专业投资者通常会问哪些问题；
- 用户希望工具能够主动引导，而不是等待一个完美问题；
- 用户接受功能与 ChatGPT 重复，但不接受花费大量时间做出一个更难用、更不可靠的替代品；
- 用户使用 Ubuntu 进行本地开发；
- 本地编程工具可以是 Codex、Claude Code 或其他满足要求的工具。

具体财务状况、持仓、风险承受能力和投资目标属于私密动态数据，不写入公开项目文档。

## 6. 产品原则

1. **Questions Before Answers**：用户不知道该问什么时，系统必须主动组织问题。
2. **Learning Before Dependence**：帮助用户形成判断能力，而不是制造权威依赖。
3. **Risk Before Return**：先检查安全边界、期限、集中度和损失承受能力。
4. **Evidence Before Confidence**：专业语气不代表可靠，重要结论必须有来源和不确定性。
5. **Personal Context with Dissent**：理解用户，但必须主动寻找反方证据，避免迎合持仓。
6. **Validate Before Automate**：未经真实使用证明的功能不进入开发。
7. **Local and Private by Default**：个人财务事实默认保存在本地私有存储。
8. **Quiet by Default**：系统应减少信息焦虑，不把每条新闻变成交易信号。

## 7. 长期领域对象

只有当真实使用证明有必要时，AIIR 才逐步维护以下对象：

- Investment Profile：目标、期限、约束和风险边界；
- Portfolio Snapshot：某一时点的资产与持仓事实；
- Research Question：用户的原始问题和系统补全的问题；
- Source：官方或高质量来源；
- Evidence：可定位、带时间的事实片段；
- Claim：可验证陈述；
- Thesis：关于某资产或策略的暂时判断；
- Decision Record：当时做出的决定及理由；
- Review：基于后来事实的复盘；
- Friction：成熟工具无法稳定解决的重复问题。

这些对象是候选，不是当前必须实现的数据库表。

## 8. 项目边界

AIIR 可以：

- 教学和解释；
- 生成研究计划；
- 寻找和总结资料；
- 讨论资产配置方案；
- 分析持仓相关风险；
- 提出支持和反对观点；
- 记录决定和复盘；
- 给出带条件、带不确定性的行动选项。

AIIR 不可以：

- 自动交易；
- 承诺收益；
- 将模型判断冒充事实；
- 在不了解用户基本约束时给出确定性仓位建议；
- 以“几十年经验专家”的身份要求用户服从；
- 仅凭价格变化制造行动压力。

## 10. 核心宣言

> AIIR 不负责假装比最先进的大模型更聪明。
> AIIR 负责让成熟模型在真实、准确、长期的个人投资上下文中，更主动、更克制、更可复盘地工作。
