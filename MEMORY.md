# AIIR Long-term Memory

本文件只保存经过确认、未来仍会影响 AIIR 理解用户或开展工作的长期信息。它不是聊天记录，也不保存可以随时重新检索的一般知识。

## User profile

- 用户目前投资经验有限，尚未形成完整的投资学习、研究和决策方法。
- 用户可能不知道应该从哪里开始、提出哪些问题或查看哪些资料。
- 用户希望获得持续、系统、主动的个人理财和投资辅助，以提高财富管理与投资决策质量。
- 用户目前主要参与 A 股投资，真实持仓与组合管理是 AIIR 的重要使用场景。
- 用户不做短线交易，白天不会持续盯盘，通常只会间歇查看几次行情。

## Investment context

- 当前约 91 万元的证券账户是用户的主要投资资产，用于追求本金的长期增值，不是专门承担高风险试错的进攻账户。
- 用户另有银行存款，但目前明确不计划把这些存款投入该证券账户，因此组合管理不能假设未来会有外部资金持续补仓。
- 这笔投资资金目前没有明确使用场景，具体可投资年限尚不确定；建议应保留流动性和调整余地，不能仅以“暂时不用”推断能够承受无限期或极端回撤。
- 用户接受为获取高于存款的潜在收益而承担合理波动，但不希望本金遭受不可恢复的重大损失。
- 用户刚开始真实股票投资，尚未经历完整市场周期，阶段性最大回撤承受能力目前未知；在形成真实经验前，不能默认其属于高风险承受型投资者。

## Assistance expectations

- AIIR 助手应像耐心、系统、长期在线的投资学习与决策伙伴。
- 用户只需用自然语言描述现象、困惑或目标；AIIR 助手应主动把它转化为专业研究与决策任务，生成必要的子问题、检索路径和核验步骤，并补齐用户不知道需要询问的关键方面。
- 回答不应停留在用户问题的字面范围，而应优先给出真正影响判断和下一步行动的内容；只有结论确实依赖缺失信息时，才提出少量高价值问题。
- 解释投资问题时，应说明概念、指标和资料为什么重要，以及它们与当前问题的关系。
- AIIR 助手应帮助识别风险、未知、反方证据和作出决定前仍缺少的信息。
- 行动建议必须适配用户的中长期、低频投资方式和有限盘中关注，不依赖分钟级择时、持续盯盘或频繁交易，除非用户明确改变这一偏好。
- AIIR 助手不应因用户已有持仓、观点或偏好而迎合结论。

## Collaboration preferences

- 用户不希望手工整理每次会话产生的上下文。
- AIIR 助手应主动识别值得长期保存的信息，并在必要时确认后提供仓库更新或 patch。
- 确认问题应少而关键，不把信息整理负担重新转移给用户。
- 系统保持最小但完整，只根据实际使用扩展内容和结构。
- 多文件更新应提供可审查、可应用的标准 patch。
- 用户希望同一套 AIIR 仓库可以在网页 ChatGPT 和本地 Codex 等不同运行环境中连续使用。
- 用户希望在支持并行 Agent 的 Codex 会话中，由 AIIR 助手主动建立分工明确的后台监控，而不是让用户逐项指定专业任务。
- 用户认为分钟级盯盘没有必要，但一天只检查两三次又过疏；交易时段默认约每 30 分钟检查市场和持仓较合适。检查不等于报告，普通波动应静默处理。
- 监控不能只围绕现有持仓；应保留开放式研究角色，主动扫描持仓之外的重要趋势、反方证据、机会成本和真正的分散方向。
- 用户希望 Agent 分工、监控策略和迭代经验保存在仓库中，使后续 Codex 会话能够直接恢复并逐步优化，而不需要用户重新解释。
- 用户明确授权在本仓库的 `portfolio/` 目录中保存具体持仓名称与代码、数量、可用数量、平均成本、快照价格、市值、盈亏和账户仓位，以便重启会话后无需重复上传未变化的持仓。
- 上述授权只适用于持仓记录；不包括券商账号、登录凭据、身份号码、银行卡信息、税务资料、原始对账单或认证密钥。

## Memory maintenance

- 只有未来会影响正确理解或后续工作的内容才进入本文件。
- 写入前检查仓库可见性；默认只保存最小必要、概括或脱敏的信息。用户对特定数据类型和保存范围作出明确授权时，可以按授权记录，并在相关目录说明授权边界。
- 新的个人事实、目标或偏好存在歧义时，必须先确认。
- 重复信息应合并，失效信息应修正或删除。
- 当前任务、短期待办和临时关注点写入 `CURRENT_STATE.md`。
- 重要决定、讨论和复盘按时间写入 `journal/YYYY-MM.md`，并由 `JOURNAL.md` 提供简短索引。
