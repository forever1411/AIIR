# AIIR 路线图

> **原则：** 路线图由验证结果驱动，不由想象中的完整平台驱动。

## Phase 0：ChatGPT-first Pilot

### 目标

不用开发软件，验证成熟大模型加结构化个人上下文是否已经足以解决主要需求。

### 交付

- 专用 ChatGPT Project；
- 系统提示；
- 投资背景、组合、规则、日志和摩擦模板；
- 三个真实场景；
- 与普通聊天的对照评估；
- 是否进入软件开发的决定。

### 退出条件

- 至少三个真实场景完成；
- 至少一个重复摩擦被确认；
- 该摩擦无法仅靠改 Prompt、模板或工作习惯解决；
- 用户愿意持续使用该流程。

### 可能结果

1. **ChatGPT 已足够**：继续使用，不开发软件；
2. **需要薄工具**：进入 Phase 1；
3. **流程本身无价值**：停止项目或重新定义问题。

三个结果都属于有效验证结果。

## Phase 1：Local Context Pack

### 触发条件

用户反复需要重新提供准确的投资背景、持仓或规则。

### 候选能力

- 本地私有文件或 SQLite；
- 投资背景和持仓的结构化维护；
- 生成当前任务所需的最小模型上下文包；
- 数据时点和完整性检查；
- 导出 Markdown、CSV 或 JSON。

### 明确不做

- Web 平台；
- 多用户；
- 自动监控；
- 多 Agent；
- 向量数据库；
- 自动交易。

## Phase 2：Decision Journal and Review

### 触发条件

聊天和手工文件无法稳定还原过去决定，或复盘成本过高。

### 候选能力

- 自然语言记录决定；
- 自动提取理由、风险、假设和失效条件；
- 保存当时可知信息；
- 计划复盘时间；
- 对比结果、过程和运气；
- 统计重复行为错误。

## Phase 3：Deterministic Portfolio Checks

### 触发条件

组合计算反复出现、手工容易出错，且对决策有明显价值。

### 候选能力

- 仓位和资产类别占比；
- 单一资产、行业、地区和币种集中度；
- 现金和流动性检查；
- 简单再平衡情景；
- 数据过期和缺失提醒。

### 限制

- 只做解释性和确定性计算；
- 不生成“最佳组合”；
- 不自动执行再平衡。

## Phase 4：Official Source Assistant

### 触发条件

寻找官方财报、公告和高质量资料成为稳定摩擦。

### 候选能力

- 按市场和标的定位官方来源；
- 发现最新财报、公告和电话会议材料；
- 记录报告期、发布时间、获取时间和原始链接；
- 去重和版本管理；
- 将资料交给成熟模型分析。

## Phase 5：Change and Thesis Review

### 触发条件

用户已有稳定研究记录，且新旧信息比较具有真实价值。

### 候选能力

- 将新证据与旧 Thesis 对比；
- 标记支持、削弱、冲突和未知；
- 只对实质变化进入注意力队列；
- 关联持仓和风险规则；
- 默认不生成交易动作。

## 延后清单

在没有明确证据前，以下内容不进入 Roadmap：

- 多 Agent 研究团队；
- 模型投票和动态路由；
- 知识图谱；
- 实时全市场行情；
- 社交媒体情绪平台；
- 自动晨报和周报；
- 券商连接；
- 自动下单；
- 公共 SaaS；
- 多租户权限系统；
- 面向机构的合规工作流。

## Roadmap 变更规则

新增阶段或功能必须同时满足：

1. 关联一条真实 friction；
2. 说明为什么成熟工具不足；
3. 给出最小验证方案；
4. 给出停止条件；
5. 不依赖尚未验证的其他大模块；
6. 由项目负责人确认。
