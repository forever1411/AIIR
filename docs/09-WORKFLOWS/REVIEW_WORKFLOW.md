# 投资复盘工作流

## 1. 目的

复盘不是用结果倒推当时是否聪明，而是评估当时在可知信息下的过程质量。

## 2. 必须恢复的资料

- 原始 Decision Record；
- 当时的 Portfolio Snapshot；
- 当时可获得的来源；
- 当时的核心假设和未知；
- 失效条件；
- 后来发生的新事实。

不得把后来信息混入“当时应该知道”。

## 3. 复盘问题

### 3.1 过程

- 当时的问题定义正确吗？
- 是否核对了可靠来源？
- 是否寻找反方证据？
- 是否考虑仓位和替代方案？
- 是否受情绪或市场噪声影响？

### 3.2 判断

- 哪些假设正确？
- 哪些假设错误？
- 哪些事实被遗漏？
- 哪些风险判断不足？
- 结论是否超过证据？

### 3.3 结果与运气

- 好结果是否来自好过程，还是偶然？
- 坏结果是否来自坏过程，还是合理风险实现？
- 是否存在不可预测事件？

### 3.4 行为

- 是否违反预先规则？
- 是否因成本价、浮盈亏或他人观点改变标准？
- 是否频繁检查价格导致不必要行动？

## 4. 输出

```text
原决定摘要
当时信息集
后来事实
过程做得好的地方
过程需要改善的地方
判断与运气
偏误
规则更新候选
下一次如何做
```

## 5. 规则更新

复盘结论不能自动修改投资规则。

规则候选至少满足：

- 不是单次结果的情绪反应；
- 能解释一个重复问题；
- 不与更高优先级安全原则冲突；
- 由用户确认；
- 保留变更原因和日期。
