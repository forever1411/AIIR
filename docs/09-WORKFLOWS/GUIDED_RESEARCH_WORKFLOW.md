# 引导研究工作流

## 1. 适用场景

当用户说：

- “我想了解这家公司”；
- “这个新闻是什么意思”；
- “我不知道该看什么”；
- “这个投资值得研究吗”；

系统不得要求用户先提出专业问题。

## 2. 步骤

### Step 1：识别目的

先判断用户是在：

- 学习概念；
- 了解公司；
- 判断新闻影响；
- 准备投资决定；
- 复盘旧决定。

目的不清时，只问一个最关键问题。

### Step 2：检查必要个人背景

只询问当前任务需要的最少信息。例如，公司基础学习通常不需要完整组合；真实买入讨论才需要期限、仓位和风险约束。

### Step 3：生成研究地图

对公司研究，通常覆盖：

1. 公司如何赚钱；
2. 客户和产品；
3. 行业与竞争；
4. 收入、利润、现金流和资产负债；
5. 管理层和资本配置；
6. 主要风险；
7. 当前市场预期和估值；
8. 哪些事实会改变判断。

不要一次性展开全部。先给地图，再逐步推进。

### Step 4：寻找来源

优先原始来源，并记录：

- 名称；
- 发布者；
- 报告期；
- 发布时间；
- 链接；
- 用途。

### Step 5：提取事实

事实必须可定位。数字检查单位、币种、口径和时间。

### Step 6：解释含义

用用户能理解的语言说明：

- 这个事实是什么；
- 为什么重要；
- 它支持或削弱什么；
- 还有什么不知道。

### Step 7：反方检查

至少回答：

- 最强反方观点是什么？
- 当前分析最可能错在哪里？
- 是否存在另一种解释？
- 如果用户没有持仓，结论会不会不同？

### Step 8：输出下一步

输出不必直接给买卖建议。可以是：

- 继续阅读某份官方材料；
- 核对某个数字；
- 比较竞争对手；
- 等待下一份数据；
- 进入决策准备流程；
- 当前无需行动。

## 3. 标准输出

```text
研究目的
当前结论摘要
已核实事实
分析解释
反方证据
未知和缺失
与个人情况的关系
下一步研究
是否具备决策条件
来源
```

## 4. 停止条件

出现以下情况应暂停结论：

- 关键来源不可访问；
- 数字冲突无法解决；
- 用户目标或期限不清；
- 重要术语尚未理解；
- 分析过度依赖单一来源；
- 用户处于强烈情绪状态；
- 问题涉及需要专业许可的复杂事项。
