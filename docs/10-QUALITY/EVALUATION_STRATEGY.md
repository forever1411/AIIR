# AIIR 评估策略

## 1. 核心问题

AIIR 是否比规范地直接使用成熟大模型更有用？

评估必须有基准线，不能只看 AIIR 输出是否“像专业报告”。

## 2. 基准

每个关键场景比较：

- Baseline：普通 ChatGPT 对话；
- AIIR V0：项目指令 + 私有上下文 + 工作流；
- 未来软件：本地控制层 + 同一成熟模型。

尽量使用相同问题、相近时间和同一来源范围。

## 3. 指标

### 3.1 启动难度

- 用户需要知道多少专业术语？
- 是否可以从一句模糊问题开始？
- 需要准备多少资料？

### 3.2 研究质量

- 是否提出关键问题？
- 是否优先使用原始来源？
- 引用是否支持对应结论？
- 是否正确处理时间、单位和口径？
- 是否区分事实和推断？

### 3.3 决策质量

- 是否检查个人目标和期限？
- 是否检查风险、集中度和替代方案？
- 是否主动寻找反方证据？
- 是否明确未知和失效条件？
- 是否避免无依据的确定建议？

### 3.4 长期价值

- 一段时间后能否恢复当时理由？
- 是否减少重复提供上下文？
- 是否支持真实复盘？
- 是否积累了可迁移的个人资产？

### 3.5 负面影响

- 是否增加信息焦虑？
- 是否诱发频繁交易？
- 是否制造对模型权威的依赖？
- 是否泄露不必要个人数据？
- 是否让流程比普通聊天更复杂？

## 4. 评分建议

每项用 0 至 3 分：

- 0：失败或明显有害；
- 1：部分满足，需要大量人工补救；
- 2：基本满足；
- 3：明显优于基准。

不计算一个掩盖问题的总分。保留分项和定性说明。

## 5. 软件立项门槛

功能进入开发至少满足：

- 在两个以上真实场景出现；
- 对时间、错误或认知负担有明显影响；
- Prompt 和模板无法稳定解决；
- 最小软件方案有清晰验收；
- 预期增益大于新增维护成本；
- 不增加不可接受的隐私或行为风险。

## 6. 禁止指标

不得单独使用以下指标证明 AIIR 有价值：

- 报告字数；
- 模型调用次数；
- Agent 数量；
- 数据源数量；
- 短期收益；
- 看起来专业；
- 用户使用时长；
- 每日提醒数量。
