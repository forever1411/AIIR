# Review 标准

## 1. 产品 Review

- 是否解决真实用户问题？
- 用户是否必须先知道专业问题？
- 是否有更简单的成熟工具方案？
- 是否关联 friction？
- 是否定义停止条件？
- 是否可能增加焦虑或过度交易？

## 2. 投资内容 Review

- 事实是否有来源？
- 来源是否为当前正确版本？
- 时间和报告期是否清楚？
- 数字的单位、币种和口径是否一致？
- 事实、推断和未知是否分离？
- 是否包含反方证据？
- 是否检查个人约束？
- 是否避免承诺或权威伪装？

## 3. 隐私 Review

- 是否使用真实个人数据？
- 数据是否进入 Git、日志或测试？
- 发给模型的数据是否最小化？
- 是否可导出和删除？
- 密钥是否隔离？

## 4. 架构 Review

- 当前阶段真的需要软件吗？
- 是否引入不必要基础设施？
- 是否保持本地优先和模型独立？
- 确定性逻辑是否与模型逻辑分离？
- 模型失败时是否可降级？
- 是否可在不使用模型时访问核心个人记录？

## 5. 代码 Review

- 变更是否最小？
- 是否有清晰验收标准？
- 是否有测试？
- 是否使用虚构数据？
- 是否处理错误和非法模型输出？
- 是否记录数据时点和来源？
- 是否避免供应商 SDK 泄漏到业务层？

## 6. Review 结论

- Approved；
- Approved with follow-up；
- Changes required；
- Rejected：不值得开发或风险超过价值。

拒绝实现是正常且有效的 Review 结果。
