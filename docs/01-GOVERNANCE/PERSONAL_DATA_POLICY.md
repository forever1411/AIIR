# 个人投资数据政策

## 1. 范围

个人投资数据包括但不限于：

- 收入、支出、现金和负债；
- 投资目标和时间期限；
- 风险承受能力；
- 账户、持仓、数量、成本和交易记录；
- 税务相关信息；
- 决策日志和行为偏误记录；
- 与家庭财务有关的信息。

## 2. 默认存储位置

真实个人数据只能存放在：

- `workspace/private/`；
- `data/private/`；
- 其他明确被 `.gitignore` 排除的本地私有位置。

仓库中只保留模板、虚构样例和 Schema。

## 3. 禁止提交

禁止提交：

- 账户号码；
- 身份证件或地址；
- API Key、Token 和密码；
- 真实持仓和成本文件；
- 银行或券商导出原件；
- 未脱敏的模型请求日志；
- 含个人财务事实的截图和报告。

## 4. 第三方模型最小化

发送给模型前：

- 删除账户标识；
- 只保留与当前问题相关的持仓；
- 能用仓位百分比时不发送完整金额；
- 能用年龄区间和期限区间时不发送精确敏感数据；
- 不发送 API Key、身份信息或无关家庭信息；
- 记录哪些类别的数据离开了本地环境。

## 5. 数据质量

个人事实必须带：

- 数据时点；
- 来源或录入方式；
- 币种；
- 是否估算；
- 最后核对时间。

模型记忆不得作为持仓、现金或成本的唯一事实来源。

## 6. 用户权利

用户必须能够：

- 查看所有保存数据；
- 修正错误；
- 导出通用格式；
- 删除部分或全部数据；
- 选择不向模型发送某类信息；
- 关闭未来自动化。

## 7. 日志

日志应记录运行状态和错误，不应默认记录完整 Prompt、完整持仓或模型响应。调试需要敏感内容时，应使用临时、受控和可删除的日志。

## 8. 备份

个人数据备份必须：

- 明确位置；
- 避免公共云误同步；
- 在可能时加密；
- 有恢复测试；
- 能被用户完全删除。
