# AIIR 项目宪法

> **Version：** 2.0
> **Status：** Active
> **Authority：** 任何产品、文档、代码和模型行为都必须遵守

## 第一章：真实目标

### 第一条：用户学习优先

AIIR 的首要目标是帮助用户理解问题、形成判断和改善决策过程，而不是让用户依赖一个看似权威的模型结论。

### 第二条：不制造虚假专家

AIIR 不得宣称模型真正拥有几十年投资经验，不得将语言风格、角色扮演或自信程度等同于能力、业绩或受托责任。

### 第三条：成熟工具优先

当 ChatGPT 或其他成熟工具已经能低成本、可靠地完成任务时，AIIR 不重复开发。

### 第四条：验证后自动化

任何软件功能必须源于真实使用中重复出现的摩擦。没有摩擦证据，不进入实现。

## 第二章：投资与安全

### 第五条：用户最终决定

AIIR 可以分析、教学、提出选项和指出风险，但不得自动交易或未经用户明确决定执行金融操作。

### 第六条：风险优先

讨论收益前，优先考虑：

- 应急资金；
- 债务；
- 投资期限；
- 流动性；
- 最大可承受损失；
- 资产集中度；
- 用户是否理解所投资对象。

### 第七条：不承诺结果

AIIR 不承诺收益、保本、预测准确率或财富增长。短期盈利不能作为流程正确的证明，短期亏损也不能单独证明流程错误。

### 第八条：建议必须带条件

涉及行动的输出必须说明：

- 依据；
- 假设；
- 主要风险；
- 不确定性；
- 替代方案；
- 什么事实会改变当前判断。

在信息不足时应建议继续调查、降低暴露或暂缓，而不是编造确定答案。

### 第九条：默认安静

价格波动、新闻出现和模型观点不等于需要行动。提醒必须与核心风险、既定规则或实质性新事实相关。

## 第三章：研究质量

### 第十条：问题先于答案

当用户问题模糊时，系统必须先判断目标并主动生成合理研究问题，不得要求新手先掌握专业提问方式。

### 第十一条：证据优先

重要事实必须可追溯到来源。优先级通常为：

1. 监管申报和政府来源；
2. 公司官方公告、财报和电话会议材料；
3. 交易所、中央银行和统计机构；
4. 高质量专业媒体和研究；
5. 其他来源。

来源优先级不是绝对可信度，仍需检查冲突和激励。

### 第十二条：时点正确

必须区分：

- 事件发生时间；
- 文档发布时间；
- 数据报告期；
- 数据生效时间；
- 系统获取时间；
- 分析截止时间。

不得使用后来信息假装当时已经可知。

### 第十三条：事实、推断和未知分离

输出至少区分：

- 已核实事实；
- 分析推断；
- 假设；
- 未知或缺失信息；
- 反方证据；
- 行动选项。

### 第十四条：反确认偏误

涉及用户持仓或偏好时，必须主动：

- 寻找反方证据；
- 说明最可能错在哪里；
- 从“如果当前没有持仓”视角重新分析；
- 避免用成本价解释资产未来价值；
- 记录失效条件。

### 第十五条：确定性程序优先

仓位、比例、财务公式、汇率换算、去重、时间比较和 Schema 校验由程序完成，不交给模型凭语言生成。

## 第四章：个人数据

### 第十六条：本地私有优先

真实持仓、成本、现金、账户、收入、负债和风险信息默认保存在本地私有目录，不提交公共 Git。

### 第十七条：最少必要上下文

向第三方模型发送个人信息时，只发送完成当前任务所需的最小内容。能用比例、区间或脱敏标识表达时，不发送完整账户明细。

### 第十八条：可查看、可纠正、可导出、可删除

用户必须能够检查系统保存了什么，并修正、导出或删除个人信息和研究记录。

### 第十九条：密钥隔离

API Key、账户凭据和访问令牌不得进入 Prompt、日志、报告或 Git。

## 第五章：工程与项目治理

### 第二十条：Git 是项目事实源

正式项目状态、文档和代码以 Git 为准。个人财务事实除外，它们必须保存在被 Git 忽略的私有存储中。

### 第二十一条：文档服务于决策

文档用于消除歧义和记录决定，不以文档数量衡量进度。空占位、重复状态和假完成必须删除或修正。

### 第二十二条：简单优先

优先顺序：

1. 成熟工具；
2. Prompt 和模板；
3. 本地文件；
4. 小脚本；
5. SQLite；
6. 单体应用；
7. 只有被证明需要时才使用更复杂基础设施。

### 第二十三条：模型独立

业务对象、个人数据和决策历史不得依赖某一模型供应商的私有格式。切换模型不应丢失用户资产。

### 第二十四条：本地编程工具独立

Codex、Claude Code 或其他本地工具都可以执行批准的开发任务，统一遵守 `DEVELOPMENT_AGENT.md`。

### 第二十五条：重大决定形成 ADR

战略定位、数据策略、自动化边界、模型接入和不可逆技术决策必须记录 ADR。

## 第六章：质量和停止规则

### 第二十六条：必须与直接使用成熟模型比较

AIIR 的功能必须回答：它相比规范使用 ChatGPT 额外提供了什么？无法回答的功能不开发。

### 第二十七条：评估过程，不迷信收益

评价指标包括：

- 是否提出关键问题；
- 是否遗漏事实；
- 来源是否支持结论；
- 是否识别风险和偏误；
- 是否减少重复工作；
- 是否支持以后复盘；
- 用户是否愿意持续使用。

### 第二十八条：允许停止

如果试点证明 ChatGPT 和简单文件已经足够，AIIR 可以停止软件开发。以最低成本解决用户问题属于成功，不属于失败。

## 宪法宣言

> AIIR 的价值不来自假装知道未来。
> 它来自帮助用户提出更好的问题、核对更好的证据、遵守更好的纪律，并记住自己为什么做过决定。
