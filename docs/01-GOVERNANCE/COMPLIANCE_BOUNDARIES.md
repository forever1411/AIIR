# AIIR 合规与产品边界（COMPLIANCE_BOUNDARIES）

> **版本：** v0.1
> **状态：** Draft
> **Owner：** Product Owner
> **最后更新：** 2026-07-22

---

## 一、目的

本文件定义 AIIR 在 MVP 阶段必须遵守的最低产品、数据和合规边界。

本文件不是法律意见。涉及正式上线、收费、跨地区用户、个性化投资建议或受许可市场数据时，必须进行专业法律与合规审查。

---

## 二、MVP 产品边界

AIIR V0.1 提供一般性的公司研究信息与证据化分析。

V0.1 不提供：

- 自动交易或自动下单；
- 代客投资决策；
- 收益承诺；
- 个性化资产配置；
- 基于用户资产、持仓或风险承受能力的“适合你”结论；
- 目标价、买入、卖出或持有评级；
- 监管意义上的投资顾问服务声明。

报告必须明确：

- 研究截止时间；
- 数据来源与限制；
- 不确定性；
- 本报告不构成个性化投资建议；
- 用户负责最终决策。

---

## 三、用户数据边界

MVP 默认不要求收集：

- 身份证件或精确住址；
- 银行或券商账户凭证；
- 完整资产负债信息；
- 详细持仓；
- 风险承受能力画像；
- 与研究无关的个人数据。

如未来功能确需处理上述信息，必须先定义：

- 明确用途；
- 用户授权；
- 最小字段；
- 访问权限；
- 加密与隔离；
- 保留期限；
- 删除机制；
- 是否允许发送给外部模型供应商。

---

## 四、数据与内容授权边界

接入任何市场数据、财报、新闻、研报或社交媒体内容前，必须记录：

- 来源；
- 许可或使用条款；
- 是否允许缓存；
- 是否允许长期保存；
- 是否允许加工；
- 是否允许向用户展示原文或摘要；
- 是否允许再分发。

公开可访问不等于可以无限制存储或再分发。

MVP 优先使用可明确验证来源和使用边界的官方资料，例如公司公告、监管披露和公司投资者关系资料。

---

## 五、模型供应商边界

接入模型 Provider 前必须确认：

- 请求数据是否被保留；
- 是否用于模型训练；
- 数据存储地区；
- 日志和删除机制；
- 企业或 API 数据政策；
- 敏感数据处理限制。

发送给 Provider 的数据必须最小化，并默认排除：

- API Key 和其他密钥；
- 用户身份信息；
- 未经授权的完整文档；
- 内部安全日志；
- 与当前任务无关的历史上下文。

---

## 六、研究输出边界

系统应区分：

- `Fact`：来源直接支持的事实；
- `Inference`：基于事实形成的推断；
- `Opinion`：观点或解释；
- `Unknown`：当前证据不足；
- `Conflict`：可靠来源之间存在冲突。

核心结论必须包含证据引用、置信度和失效条件。

引用不得只证明主题相关，而必须实质支持对应 Claim。

---

## 七、上线前最低检查

正式对外提供服务前，至少完成：

- 目标司法辖区确认；
- 用户协议与隐私政策；
- 投资建议边界审查；
- 数据授权与再分发审查；
- 模型供应商数据政策审查；
- 安全与密钥管理审查；
- 用户数据导出和删除机制；
- 报告免责声明与错误反馈机制。
