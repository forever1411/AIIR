# ADR-001：在开发软件前采用 ChatGPT-first 验证

- **Status：** Accepted
- **Date：** 2026-07-22
- **Owner：** Project Owner

## Context

AIIR 最初被定义为完整 AI 投资研究平台。项目负责人投资经验有限，希望获得类似资深财经专家的帮助，但进一步讨论发现：

- 成熟大模型已经能完成大部分单次检索、解释和分析；
- 用户真正困难之一是“不知道该问什么”；
- 从零开发完整平台可能做出比 ChatGPT 更难用、更不可靠的工具；
- 当前尚无真实使用证据证明需要多 Agent、自动监控或大型数据架构。

## Options

### A. 继续建设完整平台

优点是愿景完整；缺点是范围巨大、验证晚、重复成熟模型能力。

### B. 立即建设薄本地工具

优点是可开始编码；缺点是仍未确定最需要解决的摩擦。

### C. 先用 ChatGPT Project 和结构化文件人工运行 AIIR

优点是最快验证用户流程，并能识别真正需要软件化的环节。

### D. 完全停止 AIIR

成本最低，但可能错过长期上下文、纪律和复盘方面的真实价值。

## Decision

选择 C。

当前项目进入 Phase 0：ChatGPT-first Pilot。软件开发只有在重复 friction 被证实、且 Prompt 或模板无法解决后才开始。

## Consequences

### Positive

- 更快开始真实使用；
- 避免重做成熟模型；
- 让开发范围由证据决定；
- 即使不开发软件，也能获得可用流程和模板。

### Negative

- 项目短期看起来不像传统软件开发；
- 部分能力依赖 ChatGPT 产品限制；
- 需要用户真实记录使用体验；
- 无法立即获得自动化带来的便利。

## Validation

完成三个真实场景，对比普通 ChatGPT 和 AIIR 流程，并记录重复 friction。

## Reversal

若试点证明人工流程无法执行，但核心需求明确且薄软件可验证，可进入 Phase 1。若试点没有明显价值，则停止软件计划。
