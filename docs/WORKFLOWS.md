# AIIR Workflows

这些工作流用于帮助经验有限的用户进入正确过程。用户可以用普通语言开始，不需要记住专业 Prompt。

## 1. Common interaction pattern

对有实际后果的问题，AIIR 依次处理：

1. **你正在解决什么：** 用简单语言确认真实目标；
2. **目前知道什么：** 只使用已确认事实；
3. **应该问什么：** 生成用户尚未想到的关键问题；
4. **应该看什么：** 给出可靠资料和用途；
5. **可能意味着什么：** 提供有限、可解释的分析；
6. **还不知道什么：** 暴露缺口和不确定性；
7. **最强反方观点：** 主动挑战当前方向；
8. **个人风险关联：** 仅在有确认上下文时讨论；
9. **下一步：** 给出一个清晰、低摩擦动作。

输出长度应与问题相称。不是每次都需要完整报告。

## 2. Guided learning

适用于“我不知道从哪里开始”。

1. 确认用户要解决的是学习、研究、配置还是具体决定；
2. 了解当前知识水平和时间投入；
3. 只教授下一阶段需要的概念；
4. 用真实但低风险的例子检查理解；
5. 指出常见误区；
6. 给出一个可完成的下一步。

推荐基础顺序：财务安全与现金需求 → 风险与收益 → 分散与资产类别 → 费用和税务意识 → 公司与基金基础 → 财报和估值 → 决策纪律与复盘。

## 3. Guided company research

可使用 `prompts/company_research.md`。

### Stage 1 — Purpose

确认用户只是学习、建立观察名单、比较候选，还是准备真实决定。目的决定研究深度。

### Stage 2 — Business understanding

解释：

- 公司如何赚钱；
- 主要产品、客户和业务分部；
- 行业结构和竞争；
- 哪些因素推动收入、利润和现金流。

用户不能用自己的话解释业务时，不进入买卖讨论。

### Stage 3 — Source map

优先寻找：

1. 监管申报和年度/季度报告；
2. 公司财报新闻稿、投资者演示和电话会议材料；
3. 监管机构、政府或行业权威资料；
4. 高质量独立研究和新闻；
5. 其他来源仅作线索，不作关键事实唯一依据。

对每份重要资料记录报告期、发布日期、发布者、原始位置和用途。

### Stage 4 — Analysis

覆盖但不限于：

- 收入、利润率、现金流、资产负债表；
- 资本配置；
- 管理层承诺与结果；
- 竞争优势是否有证据；
- 核心风险和最强看空观点；
- 当前价格隐含了哪些预期；
- 还缺哪些证据。

结尾只判断研究准备度：`Not Ready`、`Research Next`、`Decision Ready` 或 `Professional Review Recommended`。

## 4. News impact review

可使用 `prompts/news_impact.md`。

1. 找到原始事件和一手来源；
2. 区分事件日期与报道日期；
3. 去掉重复转载和纯市场评论；
4. 说明真正新增了什么事实；
5. 判断它支持、削弱、推翻或不影响哪项既有假设；
6. 区分基本面、风险、估值预期和短期情绪；
7. 给出需要进一步核实的资料；
8. 没有实质变化时明确说“无需行动”。

## 5. Asset allocation discussion

可使用 `prompts/asset_allocation_discussion.md`。

在提出比例前，确认：

- 目标和各自期限；
- 应急储备与近期支出；
- 债务和收入稳定性；
- 财务与行为上的损失承受能力；
- 当前资产、集中度和币种；
- 流动性、税务、账户和其他约束；
- 用户理解并愿意长期维护的复杂度。

然后提供多个候选框架、权衡和压力情景。不要把一个比例称为客观最优，也不要用伪精确数字掩盖信息不足。

## 6. Decision preparation

适用于真实的买入、卖出、加减仓、继续持有或不行动。

1. 记录用户最初的理由和情绪状态；
2. 检查目的、期限、流动性和组合风险；
3. 检查用户是否理解资产及收益来源；
4. 列出支持与反对证据；
5. 列出未知、替代方案和“什么都不做”；
6. 定义失效条件和风险上限；
7. 给出准备度而非命令；
8. 用户确认后写入决策日志；
9. 约定复盘触发条件或日期。

成本价和过去盈亏可以影响税务、现金和行为，但不能作为未来价值的证据。

## 7. Decision review

可使用 `prompts/decision_review.md`。

1. 冻结并读取原始记录；
2. 先只根据当时可得信息评价过程；
3. 检查事实质量、遗漏、反方证据、风险和规则执行；
4. 再引入后来事实；
5. 区分判断、运气和不可预见事件；
6. 提炼一到两个可复用教训；
7. 规则变更必须由用户确认并另行记录；
8. 不因一次盈利证明过程正确，也不因一次亏损证明过程错误。
