# 个人上下文与研究记忆

## 1. 重新定义“知识库”

AIIR 不把聊天历史或向量数据库自动等同于知识。

真正有价值的长期资产是：

- 经用户确认的个人事实；
- 带来源和时点的研究事实；
- 明确版本的观点；
- 当时的决定理由；
- 后来的复盘；
- 被验证的个人规则。

## 2. 当前 V0 形式

V0 使用简单文件：

- Investment Profile；
- Portfolio；
- Watchlist；
- Investment Rules；
- Decision Journal；
- Friction Log。

这些文件比“让模型记住我”更可检查和可移植。

## 3. 记忆类型

### Personal Facts

由用户确认，包含 as-of 时间。不得由模型静默修改。

### Research Facts

必须关联 Source 和时间。新版本不覆盖旧版本。

### Interpretations

属于模型或用户分析，必须与事实分离。

### Decisions

保存当时状态，原则上追加而不是回写。

### Lessons

复盘产生的候选经验，只有用户确认后才成为正式规则。

## 4. 更新规则

- 个人事实更新需用户确认；
- 持仓更新产生新快照；
- Thesis 更新保留旧版本和变化原因；
- 决策日志不可静默篡改；
- 模型总结属于候选，未经确认不能成为事实；
- 冲突数据同时保留并标记状态。

## 5. 检索原则

未来软件检索上下文时只提供当前任务需要的信息：

- 公司研究不必发送全部家庭财务；
- 资产配置讨论需要组合摘要和约束；
- 决策复盘需要恢复当时快照；
- 新闻影响只需要相关持仓、Thesis 和规则。

## 6. 不使用向量库的默认理由

- 当前数据量小；
- 精确事实比语义相似更重要；
- Markdown、CSV 和 SQLite 已足够；
- 向量检索可能遗漏时间、版本和权威性；
- 增加基础设施并不等于更好记忆。

只有真实检索问题出现后才评估向量能力。
