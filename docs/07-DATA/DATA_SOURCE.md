# 数据来源策略

## 1. 目标

帮助没有投资经验的用户找到“应该去哪看”，同时避免把搜索结果数量误认为研究质量。

## 2. 来源层级

### 一级：原始权威来源

- 监管机构申报；
- 公司投资者关系网站；
- 公司财报、公告和电话会议材料；
- 交易所；
- 中央银行、统计机构和政府部门；
- 基金官方文件。

### 二级：高质量专业解释

- 信誉良好的财经媒体；
- 行业协会；
- 学术研究；
- 有方法披露的专业研究。

### 三级：线索来源

- 聚合网站；
- 博客；
- 社交媒体；
- 论坛；
- 模型生成的未核实内容。

三级来源可以帮助发现问题，不能单独支撑重大结论。

## 3. 来源记录最小字段

- 标题；
- 发布者；
- 来源类型；
- 原始 URL；
- 发布时间；
- 报告期或生效时间；
- 获取时间；
- 与当前问题的关系；
- 是否为原始来源；
- 是否已人工核对。

## 4. 初学者资料导航

当用户只说“我想研究一家公司”时，系统通常按顺序寻找：

1. 公司简介和业务分部；
2. 最新年度报告；
3. 最新季度报告；
4. 最新财报新闻稿；
5. 电话会议文字稿或官方材料；
6. 重大公告和监管文件；
7. 行业与竞争资料；
8. 估值所需市场数据；
9. 重要反方来源。

系统必须解释每份资料用于回答什么问题，而不只是列链接。

## 5. 时间治理

必须显示分析截止时间，并区分：

- 文档发布时间；
- 财务报告期；
- 事件发生时间；
- 后续修订时间；
- 系统获取时间。

历史复盘不得使用当时尚未公开的信息。

## 6. 数据许可

自动获取前必须检查：

- 公开访问条件；
- robots 和频率限制；
- API 条款；
- 缓存和再分发权限；
- 是否允许保存全文；
- 是否只能保存链接和摘要。

个人自用不等于可以忽略来源条款。

## 7. 冲突处理

来源冲突时：

- 优先回到原始文件；
- 检查单位、币种、口径和报告期；
- 标记修订或重述；
- 不自动选择更符合既有观点的数字；
- 无法消解时明确保留冲突。
