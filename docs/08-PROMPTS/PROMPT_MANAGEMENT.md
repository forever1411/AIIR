# Prompt 管理

## 1. Prompt 的角色

在 Phase 0，Prompt 是 AIIR 的主要产品实现。因此必须像代码一样被版本化、评估和保持克制。

Prompt 不能替代：

- 可靠数据；
- 个人事实；
- 确定性计算；
- 安全边界；
- 用户确认。

## 2. 目录

```text
prompts/
├── system/       # 长期人格和规则
├── research/     # 具体研究工作流
├── evaluation/   # 评估样例或评审提示
└── versions/     # 需要保留的历史版本
```

## 3. Prompt 元数据

每个正式 Prompt 应包含：

- 名称；
- 版本；
- 用途；
- 输入；
- 输出结构；
- 禁止行为；
- 需要的个人数据；
- 评估场景；
- 修改日期。

## 4. 设计原则

- 用户可以从模糊问题开始；
- 先判断目标，再提问题；
- 不一次性要求过多输入；
- 默认解释专业术语；
- 强制事实、推断、未知分离；
- 强制反方证据和失效条件；
- 不模拟收益保证或权威经验；
- 不因持仓迎合用户；
- 提供明确下一步；
- 涉及行动时运行安全检查。

## 5. 修改流程

1. 从真实对话发现问题；
2. 将问题写入 friction log；
3. 先尝试最小 Prompt 修改；
4. 用固定场景对比旧版和新版；
5. 检查是否引入更长、更僵化或更迎合的输出；
6. 通过后更新版本和变更说明。

## 6. 评估

Prompt 不以“听起来更专业”评估，而以：

- 是否提出关键问题；
- 是否找到合适来源；
- 是否遗漏风险；
- 是否承认未知；
- 是否给出可执行下一步；
- 是否降低用户提问门槛；
- 是否避免信息过载；
- 是否支持日志和复盘。
