# AIIR Vision

## 1. 定位

AIIR 是一个由用户长期拥有的个人投资学习、研究与决策辅助系统。

它由两部分组成：

```text
GitHub 中可版本化的个人上下文和项目规则
+
成熟大模型持续进步的检索、解释和推理能力
```

AIIR 本身不需要成为独立软件，也不需要自建一个比通用大模型更强的投资 AI。

## 2. 用户问题

用户目前投资经验有限，通常无法从一开始就提出专业、完整的问题。用户真正需要的不是只在收到好 Prompt 后回答的工具，而是一个能够主动帮助完成以下工作的长期伙伴：

- 识别当前真正需要解决的问题；
- 告诉用户从哪里开始；
- 生成用户原本不知道应该提出的问题；
- 找到并解释可靠资料；
- 帮助区分事实、推断、观点和噪声；
- 暴露风险、未知和反方证据；
- 判断做决定前还缺少什么；
- 保存重要背景、决定和教训；
- 在未来恢复当时的上下文并进行复盘。

## 3. 核心目标

AIIR 希望长期提高：

- 用户的投资知识和独立判断能力；
- 研究过程的完整性和可理解性；
- 决策前风险检查的质量；
- 对情绪、追涨、确认偏误和信息噪声的抵抗力；
- 跨会话、跨模型的研究连续性；
- 重要决定的可追溯性和复盘质量。

AIIR 的成功不是让用户无条件依赖模型，而是让用户逐渐更清楚自己在做什么、为什么做、哪里可能错。

## 4. 核心机制

### 外部长期记忆

模型的会话记忆和平台上下文可能不完整、不可审计或随工具变化。AIIR 使用 GitHub 保存经过确认的：

- 项目身份和原则；
- 用户长期背景和偏好；
- 当前目标和待办；
- 重要决定和演进；
- 已验证的合作方法和经验。

### 可替换智能引擎

AIIR 应能使用当时最适合的成熟模型。模型更换时，用户拥有的上下文和历史不应丢失。

### 启动与保存闭环

每次会话从仓库恢复上下文，并在结束时识别、确认和回写新的长期信息。Git 提供版本历史，使记忆变化可追踪、可纠正。

## 5. 明确不是什么

AIIR 不是：

- 自动交易系统；
- 收益保证工具；
- 拥有真实投资业绩的专家；
- 替用户承担责任的投资顾问；
- 为了证明项目价值而持续制造功能的软件工程；
- 要求用户先掌握专业投资术语的问答系统。

## 6. 当前阶段

当前阶段只验证最基本的闭环：

```text
能否快速恢复正确上下文
+
能否主动提供有帮助的投资学习和决策支持
+
能否低摩擦地保存新状态和经验
```

仓库结构、记录格式和工作方式将在真实使用中逐步演进，而不是一次性预先设计完整。
