# AIIR V0 产品规格

## 1. 产品形态

AIIR V0 不是独立应用。

它由以下部分组成：

- 一个 ChatGPT Project；
- 一份系统指令；
- 一组私有个人资料；
- 三条标准工作流；
- 一份 friction 日志；
- 一套评估标准。

## 2. 核心用户故事

### US-01 不知道问什么

作为没有投资经验的用户，我只想表达一个模糊目标，系统应主动判断我需要了解什么，而不是要求我先提出专业问题。

### US-02 不知道去哪找资料

系统应告诉我应优先查看哪些官方或可靠来源，并解释每种资料的用途。

### US-03 看不懂专业内容

系统应分层解释术语、数字和商业含义，并允许我从基础继续追问。

### US-04 不知道能否做决定

系统应检查个人目标、期限、风险和缺失信息，明确我当前是否已经具备决定条件。

### US-05 容易只看支持观点的信息

系统应主动提出反方证据、替代解释和失效条件。

### US-06 以后忘记为什么决定

系统应将自然语言讨论整理为可复盘的决定记录。

## 3. 功能要求

### F-01 模糊问题诊断

输入可以只有一句自然语言。输出首先说明：

- 用户可能在解决什么；
- 当前缺少哪些个人背景；
- 接下来会按什么步骤进行。

### F-02 主动研究问题

系统自动生成分阶段问题，但每次只推进最必要的下一步，避免一次性信息过载。

### F-03 来源引导

优先建议原始和官方来源，并说明：

- 文档名称；
- 发布机构；
- 报告期；
- 发布时间；
- 原始链接；
- 它回答什么问题。

### F-04 教学式解释

解释应包括：

- 简单定义；
- 与当前问题的关系；
- 常见误解；
- 一个小例子；
- 用户接下来应该看什么。

### F-05 研究输出

关键分析必须分为：

- Facts；
- Interpretation；
- Unknowns；
- Counterevidence；
- Risks；
- Next Questions；
- Possible Actions。

### F-06 决策检查

涉及资金行动时运行决策工作流，不直接跳到“买/卖”。

### F-07 日志生成

将讨论整理成候选日志，用户确认后才进入正式记录。

### F-08 摩擦记录

每次用户感到重复、混乱、不可靠或难以复盘时，生成一条 friction 候选记录。

## 4. 非功能要求

- 低门槛：用户不需要知道专业术语；
- 渐进式：每次只给可消化的下一步；
- 克制：没有重要变化时不制造行动；
- 可核查：重要事实有来源；
- 隐私：只使用当前任务必要的个人信息；
- 可移植：数据不只存在于某个模型的记忆中；
- 可停止：用户可以随时不用 AIIR 而不丢失自己的记录。

## 5. 不包含

- 实时行情；
- 自动监控；
- API 集成；
- 后端数据库；
- 自动投资建议；
- 组合优化器；
- 自动下单；
- Agent 系统；
- 完整财报解析器。

## 6. V0 验收

V0 成功需满足：

- 用户可以用一句模糊问题开始；
- 系统至少一次提出了用户此前不知道的问题；
- 至少一个重要结论有可核查原始来源；
- 至少一次明确标记“目前信息不足”；
- 至少一次主动提出反方证据；
- 至少生成一条用户认可的决策日志；
- 至少完成一次复盘；
- 用户能明确指出 AIIR 流程相对普通聊天的增益或确认没有增益。
