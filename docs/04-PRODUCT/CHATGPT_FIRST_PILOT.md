# ChatGPT-first 试点

## 1. 目的

在开发任何业务软件前，先用成熟 ChatGPT 能力、项目指令和简单私有文件人工运行 AIIR。

试点不是临时凑合，而是产品发现阶段的正式实现。

## 2. 准备

### 2.1 创建 Project

创建名为 `AIIR Personal Investing` 的 ChatGPT Project。

将 `prompts/system/personal_investment_coach.md` 作为项目指令。

### 2.2 创建私有资料

从 `data/templates/` 复制：

- `investment_profile.example.md`；
- `portfolio.example.csv`；
- `watchlist.example.csv`；
- `investment_rules.example.md`；
- `decision_journal.example.md`；
- `friction_log.example.md`。

复制到 `workspace/private/`，删除 `.example`，填写真实内容。该目录不得提交 Git。

### 2.3 最小填写原则

不要为了“完整”一次性填写所有信息。

首次只需：

- 当前最重要的财务目标；
- 大致时间期限；
- 是否有应急资金；
- 是否存在高成本债务；
- 能承受的损失范围；
- 当前最关心的一个问题；
- 三条不希望违反的投资规则。

## 3. 三个场景

### 场景 A：引导研究

输入：

> 我想了解某家公司，但不知道应该看什么。请从零带我开始。

观察：

- 是否主动提出业务、财务、竞争、估值和风险问题；
- 是否优先找官方资料；
- 是否解释术语；
- 是否给出下一步而非一次性长报告。

### 场景 B：资产配置讨论

输入：

> 我不知道怎样配置资产。不要直接给比例，先帮我判断还需要了解我哪些情况。

观察：

- 是否询问目标、期限、现金、负债和承受能力；
- 是否解释分散和风险；
- 是否区分一般教育与个性化讨论；
- 是否避免制造“最佳组合”。

### 场景 C：决定与复盘

输入：

> 我正在考虑一个真实投资决定。请按 AIIR 决策流程帮助我检查，并生成一条决策日志。

观察：

- 是否寻找反方观点；
- 是否检查仓位和替代方案；
- 是否记录未知和失效条件；
- 日志是否在未来仍可理解。

## 4. 对照方法

每个场景尽量保留两种结果：

- 普通聊天：仅提出自然问题，不加载 AIIR 指令和资料；
- AIIR 流程：使用 Project 指令、私有上下文和工作流。

比较：

- 用户准备问题的难度；
- 是否主动补全了关键问题；
- 来源质量；
- 是否考虑个人约束；
- 是否发现反方证据；
- 输出是否可行动；
- 是否可在以后复盘；
- 总时间和认知负担。

## 5. Friction 记录

出现以下情况时写入 `friction_log`：

- 重复粘贴相同资料；
- 模型忘记或混淆持仓；
- 难以找到官方来源；
- 数值计算不稳定；
- 对话和文件难以同步；
- 决策记录无法检索；
- 同一流程反复遗漏关键步骤；
- 项目文件限制导致自动任务无法使用；
- 模型过度迎合既有观点；
- 结果太长、太频繁或诱发焦虑。

不要把一次性不便立即定义为软件需求。

## 6. 试点周期

建议至少覆盖：

- 3 个真实场景；
- 2 次以上回到旧记录；
- 1 次发现模型或来源错误；
- 1 次明确决定暂缓行动；
- 1 次正式复盘。

试点不以固定天数为唯一标准，但应覆盖真实行为，而不是只做演示。

## 7. 结束评审

试点结束后回答：

1. ChatGPT 已经足够好的部分是什么？
2. 哪些摩擦至少出现两次？
3. 哪些摩擦只需改 Prompt 或模板？
4. 哪些摩擦需要本地确定性工具？
5. 软件化是否会显著改善体验？
6. 用户是否愿意每周真实使用？
7. 是否存在依赖、焦虑或过度交易风险？
8. 下一步是继续使用、开发薄工具，还是停止？
