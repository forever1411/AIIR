## Recording rules

每个条目只在确有长期价值时创建，并尽量包括：

- 日期；
- 当时要解决的问题；
- 已知事实和关键背景；
- 形成的判断或决定；
- 主要理由、风险和未知；
- 后续行动或复盘条件。

历史条目不静默改写。后续观点或事实发生变化时，通过新条目说明。

### 2026-07-23 — 第一次真实持仓讨论与工作方式校准

**要解决的问题**

用户以真实 A 股持仓开始第一次正式 AIIR 投资会话，希望了解组合有什么问题、当日为什么下跌，以及接下来应如何处理和布局。

**已知事实与关键背景**

- 用户主要参与 A 股投资；
- 用户不做短线交易，白天不会持续盯盘，只会间歇查看行情；
- 用户投资经验有限，不能被要求先提出专业问题或自行设计完整研究流程；
- 本次通过券商截图提供了真实持仓明细；用户随后明确授权将具体持仓记录在本仓库的 `portfolio/` 目录中；
- 最新已知持仓见 `portfolio/CURRENT.md`，原始日期快照见 `portfolio/snapshots/2026-07-23.md`；
- 截图时共持有 5 只 A 股，股票市值为 796,178 元、股票仓位为 86.23%，主要集中在 AI 与半导体硬件主题，表面上有多个标的，但共同风险因子较强。

**形成的判断**

- 当前首先需要解决的是组合建设和同向风险，而不是猜测下一交易日涨跌；
- “今天为什么跌”应被专业化为市场、行业、公司、资金和账户影响等分层问题，而不是寻找一个简单新闻解释；
- “明天怎么办”对长期、低频投资者通常意味着当前变化是否破坏持有逻辑、是否触发既定风险条件，以及是否需要在可执行的时间窗口调整，并不自动等同于盘中择时；
- AIIR 助手必须主动完成问题拆解、资料检索、交叉验证和反方检查，再给出用户真正需要的决策信息。

**本次摩擦与修正**

初步行动建议曾包含开盘等待、观察盘中反弹和当日减仓等短线化表达，与用户的实际交易方式不匹配。今后在提出操作建议前，必须先使用长期记忆中的投资期限、交易频率、注意力和账户角色；默认提供低频、条件式、可预先安排的方案，而不是依赖持续盯盘。

**后续行动与复盘条件**

- 下一步建立适合用户的低频组合管理框架，包括账户角色、资金期限、可接受回撤、仓位上限、持有逻辑、失效条件和复盘节奏；
- 第一项高价值未知是该账户属于主要资产还是高风险进攻账户；
- 在上述框架形成前，不基于单日下跌机械补仓，也不为了追逐市场轮动而仓促新增主题；
- 后续复盘 AIIR 是否能够在不要求用户提出专业问题的情况下，持续给出与其实际执行能力匹配的建议。

### 2026-07-24 — 第一次多 Agent 实时监控与结构优化

**要解决的问题**

用户希望 AIIR 主动调用多个 Agent 监控当日 A 股、五项持仓及相关信息。用户不要求分钟级盯盘，但认为一天只检查两三次过疏；希望后台以合理频率检查，只在有意义时报告，并将方法保存到仓库供后续会话恢复和迭代。

**监控结构的演进**

初始结构为大盘、半导体持仓、中际旭创及组合风险三个子 Agent。实际运行后发现：

- 两个持仓 Agent 都高度集中于 AI 和半导体链，存在重复；
- 原结构容易围绕已有持仓寻找信息，缺少持仓之外的开放探索和反方证据；
- Agent 数量本身不能保证全面，职责互补和主 Agent 交叉核验更重要。

随后调整为：

1. 市场状态与系统性风险 Agent；
2. 覆盖全部五项持仓的基本面 Agent；
3. 不局限于持仓的开放研究与反方侦察 Agent；
4. 主 Agent 负责调度、交叉核验、组合判断和面向用户的报告。

交易时段市场与持仓约每 30 分钟检查一次，开放研究约每 60 分钟扫描一次；午休暂停无效行情轮询，收盘及晚间复核公告。监控频率与报告频率分开，普通波动不输出。

**监控过程中的重要发现**

- 当日盘面多次在显著普跌、科技核心相对抗跌和午后修复失败之间切换。五项持仓整体没有出现显著弱于对应板块的公司级异常，因此没有把盘中普跌直接解释为持有逻辑失效，也没有给出临时交易指令。
- 中际旭创已启动 H 股全球发售，计划发行 5,450 万股并设 15% 超额配售权，预计 2026-07-30 上市。按当前已知总股本粗算，发行会使股数增加约 4.9%，若超额配售全部行使则约 5.6%。这可能带来摊薄和 A/H 估值锚压力，也会为研发、扩产和国际化补充资金；最终定价、实际发行数和上市后折溢价仍需复核。
- 美国于 2026-07-23 发布新的 Section 301 措施，并自美国东部时间 2026-07-24 起实施。中国商品原则上适用 12.5% 附加税，但官方附件豁免 `8517.62.00` 和多个 `8542` 集成电路税号。光模块和芯片持仓是否直接受影响，仍取决于公司实际报关编码、原产地、供应链和其他独立限制；不能根据政策标题直接得出利空结论。

**形成的判断**

- 适合用户的监控不是低频获取信息，而是较高频检查、低频报告和低频行动；
- 监控必须同时覆盖市场、全部持仓、组合共同风险和持仓外开放信息；
- 报告应以“是否改变中长期逻辑、风险边界、组合判断或研究优先级”为门槛；
- 后台 Agent 的原始输出由主 Agent 过滤，用户只接收通俗、可执行的结论；
- 正式方法固化在 `docs/PORTFOLIO_MONITORING_PROTOCOL.md`，后续根据真实复盘继续优化。

**后续行动与复盘条件**

- 完成当日收盘和晚间公告核查；
- 复核中际旭创 H 股最终定价、实际发行数量和 2026-07-30 上市后的 A/H 关系；
- 核实美国新贸易措施对各持仓实际产品编码和供应链的适用范围；
- 继续确认账户角色、资金期限和可接受回撤，避免监控结论与组合行动之间缺少个人约束。

### 2026-07-24 — 飞书桥安全核查与时效性研究清理

**要解决的问题**

此前仓库已经定义飞书桥的安全目标，但尚未把文档声明与实际代码逐项核对；`CURRENT_STATE.md` 中的中际旭创 H 股发行和美国 Section 301 事项也缺少明确的核验日期、下一触发点和退出条件。

**飞书桥仓库代码核查结果**

- 原实现允许通过 `FEISHU_CODEX_SANDBOX` 长期设置 `workspace-write`，与“只有 `/aiir write` 逐轮开放写权限”的目标不完全一致；现强制全局保持 `read-only`，写权限只由单轮请求传入；
- `CODEX_APP_SERVER_URL` 原先只有文档约定，没有运行时校验；现只接受无用户名和密码的本机回环 `ws://` 地址，拒绝公网、局域网监听地址和远程 `wss://`；
- 原群聊策略只要求 `@机器人`，没有群 ID 白名单；现默认关闭群聊，只有本机配置的白名单群可用，并只开放 `help`、`status` 和只读 `ask`，不在群内暴露 thread、配对、新建会话或写权限；配对码也只允许在尚无任何授权账号时完成第一次绑定；
- 原状态中的 `lastChatId` 可能被最后一次群聊覆盖，主动通知存在发往群聊的风险；现只记录最近一次已授权私聊为 `notificationChatId`，旧状态迁移时故意不继承未知聊天目标；
- 新增最近 200 条消息 ID 轻量去重：写任务在调用 Codex 前登记，优先避免重复修改；只读任务成功后登记，临时失败时仍可重试；
- 飞书 SDK 日志降为警告级，常见凭证和会话标识在本机日志中脱敏；启动日志不再打印 thread ID，聊天错误回复不回显底层异常详情；
- 锁定依赖中的 `ws` 为 8.21.1，高于已知拒绝服务问题的修复版本 8.21.0；但本次没有访问实际运行主机或 npm 注册表，不能据此认定本机安装依赖、systemd 服务、私密配置与日志权限已经通过安全认证。

**本机仍需验证**

应用更新后，在实际运行机器执行语法检查、测试、依赖审计、健康诊断，并核对 systemd 服务和私密文件权限。升级后的主动通知在收到一次已授权私聊前会保持关闭，这是防止旧群聊目标被静默继承的有意设计。

**美国 Section 301 事项核验**

- 2026-07-23 发布的强迫劳动相关 Section 301 最终行动，是覆盖多个经济体的独立措施；中国和香港来源商品原则上适用 12.5% 税率，但官方附件列出的豁免包括 `8517.62.00` 和 `8542.31.00`、`8542.32.00`、`8542.33.00`、`8542.39.00`、`8542.90.00` 等通信及集成电路税号；
- 2025-12-23 发布的中国半导体产业专项 Section 301 行动是另一项措施，覆盖部分 `8541`、`8542` 税号，初始新增税率为 0，并计划于 2027-06-23 调升，具体幅度另行公告；
- 因此，政策标题本身不足以证明五项持仓都受到新的直接关税冲击。是否影响某家公司，仍需核对实际出口美国的产品、生产和出口原产地、报关税号、客户合同承担方式，以及出口管制等其他独立限制。组合级笼统待办已关闭，只有逐股审计出现具体证据时才重新打开。
