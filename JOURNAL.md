# AIIR Journal

本文件按时间追加 AIIR 的重要讨论、决定和演进。它保留“当时如何理解和决定”的历史，不作为当前状态的替代品。

---

## 2026-07-23 — 从软件项目转向 GitHub 驱动的认知系统

### 背景

AIIR 最初被设想为一个大而全的 AI 投资研究平台，希望像拥有几十年经验的财经专家一样分析经济、新闻、公司业绩和资产配置。

随着讨论深入，逐渐确认：成熟大模型已经能够完成大量检索、解释、研究和推理工作。真正长期缺失的是与用户相关、跨会话可恢复的个人上下文、历史决定、工作原则和经验。

### 关键决定

- 不再开发 AIIR 独立本地程序。
- 使用 GitHub 保存 AIIR 的正式上下文、规则、状态、记忆和历史。
- 使用 ChatGPT 等成熟大模型提供研究、解释、推理、质疑和交互能力。
- 用户以后只需要求模型读取 `AI_BOOTSTRAP.md`，即可恢复项目身份和当前状态。
- 每次重要会话结束时，由模型主动检查需要固化的信息；对不确定内容先确认，再提供 patch。

### 关键反思

- AIIR 不应该要求投资新手先知道如何提出专业问题。
- AIIR 不能被包装成真实拥有几十年经验且能可靠创造收益的权威。
- 先前多次陷入过度设计：在没有真实运行前建立了过多架构、目录、Prompt 和模板。
- 一度又把“最小系统”误解为只保留极少文件，忽略了 Vision、角色和交互规则同样是恢复系统身份的必要内容。

### 新方向

建立一个最小但完整的 AIIR 操作系统内核：

```text
身份与愿景
+ 模型角色
+ 基本原则
+ 启动和保存协议
+ 当前状态
+ 长期记忆
+ 历史日志
```

今后只在真实需求第一次出现时增加对应文件或目录。
