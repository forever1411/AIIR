# AIIR Changelog

本文件只记录会改变仓库结构、AIIR 身份或正式运行协议的变更。普通投资讨论和个人决定记录在 `JOURNAL.md` 或后续按需创建的专题文件中。

## 2026-07-23 — User-authorized portfolio persistence

- 新增 `portfolio/` 目录，分别保存最新已知持仓和按日期归档的历史快照。
- 在用户明确知晓仓库公开可见的情况下，按其授权记录具体持仓、数量、成本、市值、盈亏和仓位，使后续会话无需重复上传未变化的持仓。
- 将隐私规则从绝对禁止精确财务数据改为默认最小化、用户限定范围内可显式授权；账号、凭据、身份号码、银行卡信息、原始对账单和认证密钥仍排除在仓库之外。
- 定义持仓记录的启动读取、事实来源、更新、历史保护和数据时效规则。

## 2026-07-23 — First live validation and intent-to-research workflow

- 记录第一次真实 A 股持仓讨论的验证结果、主要摩擦和下一阶段目标。
- 将“用户自然语言问题 → 专业研究与决策任务”的转换明确为 AIIR 正式原则和会话协议，而不再只作为一般性的主动引导要求。
- 要求行动建议适配用户的投资期限、交易频率、可投入注意力和实际执行能力。
- 对非短线用户，默认采用条件式行动、仓位边界和定期复盘，不依赖开盘抢跑、分钟级择时、持续盯盘或频繁交易。

## 2026-07-23 — Repository coherence and safety hardening

- 区分当前会话工作事实与已经提交的正式上下文，并按信息类型明确文件职责。
- 要求远程启动和 patch 基于同一 ref 或 commit 快照，避免混用缓存页面与不同版本内容。
- 为公开仓库增加个人与财务数据最小化、概括化和脱敏边界。
- 明确外部资料只作为证据，不具有改变 AIIR 规则或权限的权力。
- 将当前状态从笼统的“可运行”收紧为“准备进行第一次真实验证”，移除与长期记忆重复的用户画像，并增加最小验证信号。

## 2026-07-23 — Tool-neutral AIIR Assistant

- 将运行角色统一定义为工具中立的 AIIR Assistant。
- 将角色文档重命名为 `docs/AIIR_ASSISTANT_ROLE.md`。
- 新增 `AGENTS.md`，作为 Codex 等本地 AI 代理的自动启动入口。
- 明确 AIIR 会话默认处于讨论模式，仓库修改需要用户明确授权。
- 明确网页 ChatGPT、本地 Codex 和其他成熟模型可以基于同一仓库连续运行 AIIR。

## 2026-07-23 — Initial baseline

- 定义 AIIR 为 GitHub 驱动的个人投资学习、研究与决策辅助系统。
- 定义用户、大模型和 GitHub 仓库之间的职责分工。
- 建立 Vision、模型角色、基本原则和交互协议组成的系统内核。
- 建立启动恢复、当前状态、长期记忆、历史记录和会话收尾机制。
- 规定新的文件、目录和流程只根据真实使用需要创建。
