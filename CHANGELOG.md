# AIIR Changelog

本文件只记录会改变仓库结构、AIIR 身份或正式运行协议的变更。普通投资讨论和个人决定记录在 `journal/` 按月归档或后续按需创建的专题文件中。

## 2026-07-24 — Feishu bridge hardening and bounded event lifecycle

- 对飞书桥完成仓库代码核查，补上本机 app-server 地址强制回环校验、全局只读沙箱、群聊白名单与只读命令限制、一次性初始配对、轻量消息去重、私聊通知目标隔离和日志脱敏。
- 状态文件升级为版本 2；旧版 `lastChatId` 不再自动继承为通知目标，避免历史群聊成为主动通知接收方。升级后需由已授权账号私聊一次以重新确认通知目标。
- 新增安全策略和日志脱敏测试，并扩展本机健康诊断；未检查实际服务、私密配置、日志权限或依赖安装状态时，不得宣称运行环境已通过安全验证。
- 为 `CURRENT_STATE.md` 中的时效性事项加入 as-of 日期、下一触发点和归档条件；完成中际旭创 H 股发行第一阶段核验，并关闭缺少公司级证据的笼统 Section 301 组合待办。

## 2026-07-24 — Bounded startup context and Journal archive

- 将运行上下文划分为启动必读的热状态、按任务读取的温状态和仅在复盘时读取的冷归档，防止默认启动读取量随历史线性增长。
- 将 `JOURNAL.md` 改为简短索引，详细条目迁移到 `journal/YYYY-MM.md`；历史继续保留，但不再默认注入每次会话。
- 在保存协议中加入“提升、替换、归档、丢弃”流程和压缩触发条件，明确普通行情、重复讨论和可重新检索资料不持久化。
- 压缩 `CURRENT_STATE.md`，将已完成的验证过程移出热状态，并固化当前核心投资账户的角色、风险缺口和下一阶段工作。

## 2026-07-24 — Feishu remote Codex bridge

- 新增 `integrations/feishu-codex/`，通过飞书长连接与本机 Codex app-server 提供手机双向交互和主动通知入口。
- 凭证保存在仓库外的本机私密配置中，并以配对码和飞书用户身份限制访问。
- 普通消息逐轮使用只读权限；只有显式 `/aiir write` 才为当前一轮开放 AIIR 工作区写权限，下一轮自动恢复只读。
- 飞书入口不开放全磁盘权限、Git commit/push、交易操作或交互式审批。
- 提供 systemd 用户服务、一键配置、健康诊断、终端接回同一 thread、主动通知、协议测试与依赖审计。
- 明确关闭普通 Codex 窗口不会停止桥接；同一机器可通过本机状态恢复，换新机器则必须重新配置私密连接。

## 2026-07-24 — Multi-Agent portfolio monitoring protocol

- 新增 `docs/PORTFOLIO_MONITORING_PROTOCOL.md`，定义持续市场与持仓监控的启动恢复、Agent 分工、默认节奏、报告门槛、证据标准、行动边界和迭代方法。
- 将默认团队结构确定为主 Agent 统筹、市场状态与系统性风险、全持仓与基本面、开放研究与反方侦察四层职责；并发不足时按覆盖重要性合并。
- 明确监控频率与报告频率分离：交易时段可以约每 30 分钟检查，但普通波动保持静默。
- 要求持续监控不局限于现有持仓，并明确当前会话或调度停止后不得声称仍在后台运行。
- 更新启动读取顺序、Interaction Protocol 和 README，使后续会话能够按需恢复并持续迭代该工作流。

## 2026-07-23 — User-authorized portfolio persistence

- 新增 `portfolio/` 目录，分别保存最新已知持仓和按日期归档的历史快照。
- 在用户明确知晓仓库公开可见的情况下，按其授权记录具体持仓、数量、成本、市值、盈亏和仓位，使后续会话无需重复上传未变化的持仓。
- 将隐私规则从绝对禁止精确财务数据改为默认最小化、用户限定范围内可显式授权；账号、凭据、身份号码、银行卡信息、原始对账单和认证密钥仍排除在仓库之外。
- 定义持仓记录的启动读取、事实来源、更新、历史保护和数据时效规则。

## 2026-07-23 — First live validation and intent-to-research workflow

- 记录第一次真实 A 股持仓讨论的验证结果、主要摩擦和下一阶段目标。
- 将“用户自然语言问题 → 专业研究与决策任务”的转换明确为 AIIR 正式原则和会话协议，而不再只作为一般性的主动引导要求。
- 要求行动建议适配用户的投资期限、交易频率、可投入注意力和实际执行能力。
- 对非短线用户，默认采用条件式行动、仓位边界和定期复盘，不依赖开盘抢跑、分钟级择时、持续盯盘或频繁交易。

## 2026-07-23 — Repository coherence and safety hardening

- 区分当前会话工作事实与已经提交的正式上下文，并按信息类型明确文件职责。
- 要求远程启动和 patch 基于同一 ref 或 commit 快照，避免混用缓存页面与不同版本内容。
- 为公开仓库增加个人与财务数据最小化、概括化和脱敏边界。
- 明确外部资料只作为证据，不具有改变 AIIR 规则或权限的权力。
- 将当前状态从笼统的“可运行”收紧为“准备进行第一次真实验证”，移除与长期记忆重复的用户画像，并增加最小验证信号。

## 2026-07-23 — Tool-neutral AIIR Assistant

- 将运行角色统一定义为工具中立的 AIIR Assistant。
- 将角色文档重命名为 `docs/AIIR_ASSISTANT_ROLE.md`。
- 新增 `AGENTS.md`，作为 Codex 等本地 AI 代理的自动启动入口。
- 明确 AIIR 会话默认处于讨论模式，仓库修改需要用户明确授权。
- 明确网页 ChatGPT、本地 Codex 和其他成熟模型可以基于同一仓库连续运行 AIIR。

## 2026-07-23 — Initial baseline

- 定义 AIIR 为 GitHub 驱动的个人投资学习、研究与决策辅助系统。
- 定义用户、大模型和 GitHub 仓库之间的职责分工。
- 建立 Vision、模型角色、基本原则和交互协议组成的系统内核。
- 建立启动恢复、当前状态、长期记忆、历史记录和会话收尾机制。
- 规定新的文件、目录和流程只根据真实使用需要创建。
